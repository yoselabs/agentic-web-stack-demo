# Agentic Web Stack

Monorepo: TanStack Start (frontend) + Hono (API server) + tRPC + Prisma + PostgreSQL + Better-Auth.

## Structure

- `apps/web/` ‚Äî TanStack Start (Vite SSR) on port 3000
- `apps/server/` ‚Äî Hono API server on port 3001
- `packages/api/` ‚Äî tRPC router + context (shared types)
- `packages/auth/` ‚Äî Better-Auth config
- `packages/db/` ‚Äî Prisma schema + client
- `packages/env/` ‚Äî @t3-oss/env-core validated env vars
- `packages/ui/` ‚Äî shadcn/ui components
- `e2e/` ‚Äî playwright-bdd (Gherkin specs + step definitions)

Each directory with a CLAUDE.md has area-specific guidance. Read it before working there.

## Commands

- `make setup` ‚Äî zero-conf: installs deps, starts Postgres, pushes schema, installs pre-commit hooks
- `make dev` ‚Äî start both web and server
- `make check` ‚Äî full quality gate: `agent-harness lint` + `tsc -b`
- `make fix` ‚Äî auto-fix lint issues
- `make test` ‚Äî BDD tests (separate test DB on port 5433)
- `make db-push` ‚Äî push Prisma schema to database
- `make db-generate` ‚Äî regenerate Prisma client

Pre-commit hooks run `agent-harness fix` then `agent-harness lint` automatically.
Never truncate lint or test output ‚Äî read the full error.

## Development Workflow (BDD-first)

When adding features, write the Gherkin spec BEFORE implementation:

1. **Write Gherkin feature file** ‚Äî define the behavior contract (will fail)
2. **Write step definitions** ‚Äî implement test helpers
3. **Implement backend** ‚Äî tRPC routes with unit/integration TDD (Vitest)
4. **Implement frontend** ‚Äî UI components
5. **Run BDD tests** ‚Äî watch them go green (`make test`)
6. **Quality gate** ‚Äî `make check` must pass before claiming done

The Gherkin spec is the source of truth for "what does done look like." Never save BDD tests for last.

## Package Naming

All workspace packages use `@project/*` prefix (e.g., `@project/api`, `@project/db`).

## Critical Rules

- **Never use `verbatimModuleSyntax` in apps/web** ‚Äî causes server bundle leaks in TanStack Start
- **Always use `import type` for AppRouter** ‚Äî value imports bundle the server into the client
- **One `initTRPC.create()` call** ‚Äî in `packages/api/src/trpc.ts` only
- **QueryClient must be per-request on server** ‚Äî see `getQueryClient()` in `apps/web/src/router.tsx`
- **TanStack Start is NOT Next.js** ‚Äî use `createServerFn`, not `getServerSideProps` or `"use server"`
- **Run `make check` before claiming work is done** ‚Äî typecheck + lint must pass

## Generated Files (do not edit)

- `apps/web/src/routeTree.gen.ts` ‚Äî auto-generated by TanStack Router plugin on `vite dev`
- `e2e/.features-gen/` ‚Äî auto-generated by `bddgen` from Gherkin feature files
- `node_modules/` ‚Äî managed by pnpm

## Common Mistakes

| Mistake | Why it breaks | Fix |
|---------|--------------|-----|
| Import `appRouter` value in client | Bundles entire server into browser | Use `import type { AppRouter }` |
| Create multiple `initTRPC.create()` | Type mismatches between routers | Single instance in `packages/api/src/trpc.ts` |
| Add `verbatimModuleSyntax: true` to apps/web | Server code leaks into client bundle | Explicitly set to `false` in apps/web/tsconfig.json |
| Edit `routeTree.gen.ts` | Overwritten on next `vite dev` | Edit route files in `src/routes/` instead |
| Use `navigate()` during React render | React setState-in-render warning | Use `useEffect` for conditional navigation |
| Hardcode localhost URLs | Breaks in non-local environments | Use env vars: `VITE_API_URL`, `CORS_ORIGIN`, `BETTER_AUTH_URL` |
| Create QueryClient as module singleton | Leaks data between SSR requests | Use `getQueryClient()` pattern (per-request on server) |
| Skip `prisma generate` after schema change | Stale types, runtime errors | Run `make db-push` (includes generate) |
| `Link to` rejects not-yet-created routes | TanStack Router types from `routeTree.gen.ts` | Use `to={"/path" as string}` temporarily, remove once route exists and `make dev` regenerates |
| `//` in JSX text content | Biome `noCommentText` flags as comment | Wrap in expression: `<p>{"https://example.com"}</p>` |
